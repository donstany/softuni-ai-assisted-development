# AI-Assisted Development — Exam Requirements

> SoftUni course: **AI-Assisted Development**
> Deliverable: a **Google Drive document** describing a small software project built with AI coding tools (GitHub Copilot, Claude Code, Augment Code, Cursor, or a combination).

---

## 1. Overview

Create a Google Drive document that describes a small software project you built using one or more AI coding assistants:

- GitHub Copilot
- Claude Code
- Augment Code
- Cursor
- Or any combination of the above

---

## 2. Choose Your Own Project Topic

- Pick any small software system: script, automation, API, mini-app, CLI tool, etc.
- Describe the **idea** and the **system requirements**.
- **Length:** up to **1 page**.

---

## 3. System Architecture — Modules

Use AI to break the system down into several **technological modules**, for example:

- Authentication
- Data processing
- UI
- Database layer
- Automation step
- Testing layer
- …

For each module, describe **your approach** and **how AI assisted you**.

---

## 4. Development Process per Module

For **every module**, include:

- **Approach & reasoning** — how you planned the solution
- **Step-by-step workflow** — what you asked the tool to generate or modify
- **Testing strategy** — how you validated correctness
- **AI tool choice** — which tool you used (Copilot, Claude Code, Augment, Cursor, etc.) and **why**
- **Key prompts or interactions** — 2–3 examples are enough

**Length:** keep the description for each module **up to half a page**.

---

## 5. Challenges & Tool Comparison

Include a section explaining:

- The **biggest challenges** you encountered
- Which tool helped you the most and **why**
- What you would **improve** if you continued working on the project

---

## 6. Working System Evidence

- Include **at least two screenshots** of the functioning system.
- Acceptable evidence: terminal output, UI, logs, Postman result, or anything that proves the implementation works.

---

## 7. Repository

- Provide a link to your **GitHub repository** containing the source code inside the document.

---

## Submission Format

| Item | Requirement |
|------|-------------|
| Format | Link to a **Google Drive document** |
| Sharing | Must be shared — **anyone with the link can view** |
| Total size | Typically **3–6 pages** |
| Repository link | Required inside the document |
| Screenshots | At least **2** of the working system |

---

## Quick Checklist

- [ ] Project idea & requirements (≤ 1 page)
- [ ] System broken into technological modules
- [ ] Per-module: approach, workflow, testing, tool choice, prompts (≤ ½ page each)
- [ ] Challenges & tool comparison section
- [ ] ≥ 2 screenshots of the working system
- [ ] GitHub repository link included
- [ ] Google Drive doc shared as "Anyone with the link can view"
- [ ] Total length ~3–6 pages
